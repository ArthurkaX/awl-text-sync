# Working Rules

This workspace contains editable STEP 7 AWL text generated by `awl-text-sync`.

## Block Identity

- One block equals one file.
- The block type and block number are immutable unless explicitly requested.
- The filename must match the block header.
- Symbol names must stay synchronized with `Project/Symbols/*.sdf`.

## Editing

- Edit only `Project/Blocks/` unless the task explicitly requires symbol changes.
- Preserve original formatting.
- Make the smallest change that solves the task.
- Do not edit files under `Build/`.

## STL Safety

- Do not mix symbolic and absolute DB addressing in one reference.
- Keep `P#...` pointer literals absolute.
- Validate indirect addressing syntax carefully.
- Jump labels must exist in the same block and must not be duplicated.

## CALL Safety

- `FB` calls require an instance DB or valid local instance.
- `FC` calls do not use an instance DB.
- Actual parameters must match the target block interface.
- Do not invent parameter names.

## Goal

Neither a human nor an AI assistant should break the project with a random rename or a syntax error.
